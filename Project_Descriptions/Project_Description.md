## 1. 프로젝트 한눈에 보기
AI_SYS는 경찰 공무원 시험 준비 과정에서 필요한 판례를 빠르게 탐색하고, 근거 중심으로 학습할 수 있도록 지원하는 AI 학습 시스템을 목표로 한다.  
현재 저장소에는 초기 기획 문서가 정리되어 있으며, 이후 RAG 기반 판례 검색, 요약, 문제 풀이 연계, 오답 복습 기능을 단계적으로 구현하는 방향을 전제로 한다.

문제 정의 정렬 요약 (현재 README 기준)
- 핵심 문제: 수험생이 판례를 찾고 확인하는 데 시간이 많이 들고, 일반 LLM 사용 시 부정확한 답변이나 할루시네이션 위험이 존재한다.
- 사용자 페인포인트: 판례 검색 반복, 학습 흐름 단절, 최신성 확인 부담, 신뢰 가능한 근거 확인 어려움.
- 제품 방향: 판례 검색, 핵심 요약, 기출 연계, 오답 기반 반복 학습을 하나의 흐름으로 묶는 학습 지원 시스템.

현재 문서화 범위
- 문제 정의와 대상 사용자 정리
- 경쟁 서비스 및 차별점 분석
- 핵심 기능과 기술 방향 정의
- 기대효과와 15주 개발 일정 초안 정리

목표 구현 범위
- 판례 검색 및 요약 인터페이스
- 근거 출처가 포함된 질의응답 기능
- 기출 문제/예상 문제 연계 기능
- 오답 이력 기반 취약 쟁점 복습 기능
- 학습 대시보드 및 개인화 복습 화면

## 2. 제품 구조

### 2-1. 사용자 경험 레이어
- 키워드 입력, 문제 지문 사진 촬영(OCR), 판례 번호 직접 입력 기반 판례 검색 화면
- 판례 요약 카드, 시험 포인트, 유사 판례 시각화
- 기출 문제 연결 및 오답 노트 확인 화면
- 개인별 취약 쟁점 복습 대시보드 (홈 화면)

### 2-2. AI 및 검색 레이어
- 판례 원문/요약 데이터 수집, 내부 DB 적재, 전처리
- 임베딩 기반 유사 판례 검색
- RAG 기반 답변 생성 및 출처 반환
- 필요 시 reranking을 통한 검색 정확도 개선

### 2-3. 서비스/API 레이어
- 사용자 질의 수신 및 검색 결과 반환
- 판례 요약, 출처, 관련 문제 데이터 응답
- 오답 이력 저장 및 복습 추천 로직 제공
- 로그 수집과 품질 점검 기능 운영

### 2-4. 현재 저장소 자산
- [README.md](../README.md)에 문제 정의, 대상 사용자, 경쟁 서비스, 개발 일정이 정리되어 있다.
- 현재 저장소는 구현 코드보다 기획과 문서 정합성을 우선하는 초기 단계 자산 중심 구조다.

### 2-5. 데이터 운영 레이어
- 외부 공개 API 존재 여부와 무관하게 서비스 기준 데이터는 내부 판례 DB에 적재해 운영한다.
- 내부 DB는 판례 원문, 메타데이터, 검색용 청크, 키워드, 유사 판례 관계 정보를 함께 관리한다.
- OCR 입력은 원문 수집 수단이 아니라 내부 판례 DB 검색을 위한 입력 보조 수단으로 사용한다.
- 검수 완료된 판례 데이터만 사용자 응답의 기준 데이터로 공개한다.

## 5. 시각화: 제품 파이프라인

```mermaid
flowchart LR
  U[수험생 질의 입력] --> S[판례 검색]
  S --> R[RAG 기반 요약 생성]
  R --> C[근거 출처와 시험 포인트 제공]
  C --> Q[관련 문제 풀이]
  Q --> W[오답 기록]
  W --> P[개인화 복습 추천]
```

## 6. 운영 및 품질 관점
- 답변에는 가능한 한 판례 출처를 함께 제공해 신뢰도를 확보한다.
- 검색 정확도와 요약 품질은 사용자 테스트와 시나리오 검증을 통해 점진적으로 개선한다.
- 비전공자 사용자도 빠르게 사용할 수 있도록 검색에서 복습까지의 흐름을 단순하게 유지한다.

## 7. 최종 수용 기준
- 문서 전반이 경찰 고시 판례 학습 지원이라는 핵심 목적을 일관되게 반영한다.
- 현재 저장소의 실제 상태가 초기 기획 단계라는 점을 명확히 설명한다.
- 사용자 여정, 기능 구조, 기대 산출물 간의 연결이 자연스럽고 모순이 없다.
